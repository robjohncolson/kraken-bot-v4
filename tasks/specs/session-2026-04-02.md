## Batch A: Ordermin Enforcement (Goal 1) — BLOCKING

### Problem
Rotation planner allocates children using `min_position_usd` (config, default $10), but Kraken rejects orders below pair-specific `ordermin` quantities. Currently `grid/sizing.py` has a hardcoded map for 7 USD pairs only. Rotation tree needs ALL pairs (AUD, ETH, BTC quotes too).

### Solution
1. **Fetch ordermin from Kraken AssetPairs API** at startup + periodic refresh (24h TTL)
   - `GET https://api.kraken.com/0/public/AssetPairs` returns `ordermin` field per pair
   - Parse into `Dict[NormalizedPair, Decimal]`
   - Cache in-memory with timestamp; persist to SQLite `pair_metadata` table for cold starts

2. **New module `exchange/pair_metadata.py`**:
   ```python
   class PairMetadataCache:
       async def refresh(self) -> None  # fetch from API, update cache + SQLite
       def ordermin(self, pair: str) -> Decimal | None  # lookup
       def meets_minimum(self, pair: str, quantity: Decimal) -> bool
   ```

3. **Enforce in rotation planner** (`trading/rotation_planner.py`):
   - In `compute_child_allocations()`: after computing `allocated_qty`, convert to base via reference price, check against `ordermin`. Skip candidate if below minimum.

4. **Enforce in rotation tree helpers** (`trading/rotation_tree.py`):
   - In `entry_base_quantity()`: validate result >= ordermin, raise `OrderBelowMinimumError` if not
   - In `exit_base_quantity()`: same check for exits

5. **Defensive check in order gate** (`exchange/order_gate.py`):
   - Final guard in `build_order_payload()` before sending to exchange

6. **SQLite migration**: Add `pair_metadata` table:
   ```sql
   CREATE TABLE IF NOT EXISTS pair_metadata (
       pair TEXT PRIMARY KEY,
       ordermin TEXT NOT NULL,
       lot_decimals INTEGER,
       updated_at TEXT NOT NULL DEFAULT (datetime('now'))
   );
   ```

7. **Remove hardcoded map** in `grid/sizing.py` — replace with `PairMetadataCache` lookup

### Affected files
- `exchange/pair_metadata.py` (NEW)
- `trading/rotation_planner.py` (edit)
- `trading/rotation_tree.py` (edit)
- `exchange/order_gate.py` (edit)
- `persistence/sqlite.py` (migration)
- `grid/sizing.py` (refactor to use cache)
- `runtime_loop.py` (inject cache, refresh on startup)
- `core/config.py` (add PAIR_METADATA_REFRESH_HOURS default)

## Batch C: Rotation Event Observability (Goals 2, 3, 4) — INDEPENDENT

### Problem
TP/SL exits, fill timeouts, and P&L are logged to stdout but not structured/surfaced in dashboard/TUI.

### Solution: Rotation Event Log
1. **New dataclass `RotationEvent`** in `core/types.py`:
   ```python
   @dataclass(frozen=True)
   class RotationEvent:
       timestamp: datetime
       node_id: str
       event_type: str  # "tp_hit", "sl_hit", "entry_timeout", "exit_escalation", "fill", "expiry"
       pair: str
       details: dict  # flexible payload (prices, quantities, reasons)
   ```

2. **Emit events in runtime_loop.py**: At each TP hit, SL hit, timeout, fill, expiry — append to `self._rotation_events: deque[RotationEvent]` (capped at 100)

3. **Include in SSE payload**: Add `rotation_events` to dashboard state
   ```python
   "rotation_events": [event.to_dict() for event in self._rotation_events]
   ```

4. **API endpoint**: `GET /api/rotation-events` returns recent events

5. **TUI integration**: Add event summary to rotation tree screen footer:
   - "Last TP: BTC/AUD +3.2% 5min ago | Last SL: ETH/AUD -1.8% 12min ago"
   - Or a mini event log below the tree table

6. **Rotation stats in dashboard**:
   ```python
   rotation_stats: {
       tp_count: int, sl_count: int, timeout_count: int,
       expiry_count: int, avg_hold_hours: float
   }
   ```

### Affected files
- `core/types.py` (add RotationEvent)
- `runtime_loop.py` (emit events at each trigger point)
- `web/routes.py` (add endpoint + include in SSE)
- `tui/screens/rotation_tree.py` (add event display)
- `tui/state.py` (parse rotation events)

### Test plan

## Batch D: Parameter Tuning + Broker Docs (Goals 5, 6) — INDEPENDENT

### D1: Parameter Tuning Validation

**Add bounds validation** in `core/config.py`:
- `ROTATION_TAKE_PROFIT_PCT`: must be > 0, warn if < 1.0 (fees eat it) or > 10.0
- `ROTATION_STOP_LOSS_PCT`: must be > 0, warn if > 5.0 (too loose)
- `MIN_BELIEF_CONFIDENCE`: must be 0.0-1.0
- `ROTATION_ENTRY_FILL_TIMEOUT_MIN`: must be > 0, warn if < 5
- `ROTATION_EXIT_FILL_TIMEOUT_MIN`: must be > 0, warn if < 1

Log warnings at startup for out-of-range values. Don't fail — operator might have reasons.

**Affected files**:
- `core/config.py` (add `validate_settings()` function)
- `runtime_loop.py` (call `validate_settings()` at startup)
